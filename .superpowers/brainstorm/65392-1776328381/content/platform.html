How should you share your design with the designer?

Each option has different trade-offs for visual richness, collaboration, and effort to set up. Pick the one that fits best — or ask about any of them.

🎨

A — Figma Design Spec

Claude creates a Figma file using the Figma MCP: color swatches, type scale, icon previews, spacing, and annotated screenshots of the live site. Designer opens it in her native tool and leaves comments inline.

Pros

  • Designer's native environment
  • Inline comments on specific elements
  • Can export to PDF/share link

Cons

  • Requires Figma account (free is fine)
  • Figma write access must be connected to Claude
  • Moderate setup effort

Model: Sonnet (sufficient) · Context: Claude Code (repo access)

🌐

B — Living HTML Design Brief

Claude adds a /design-brief/ page to your Astro site — deployed on Netlify. Shows actual rendered colors, real Lucide icons, type scale, and live links. Always up-to-date with the codebase.

Pros

  • No extra accounts needed
  • Shows actual design tokens (real CSS)
  • Accessible at a URL — easy to share
  • Fastest to build

Cons

  • Designer must annotate via screenshots + notes (no inline comments)
  • Less "native" designer workflow

Model: Sonnet · Context: Claude Code (repo access)

📄

C — PDF / DOCX Document

Claude generates a rich PDF or Word document from your design.md — with color swatches, type examples, Lucide icon previews, and screenshots. Designer annotates it with Acrobat or Word track changes.

Pros

  • Universal — works for everyone
  • Easy to email/share
  • No setup required

Cons

  • Static — won't update with code changes
  • Icons/fonts may not render perfectly
  • Less visual than Figma or live HTML

Model: Sonnet · Context: Claude Code or any

My lean: Option B (HTML Design Brief) is the fastest, most accurate, and lets the designer see exactly what's in the code — real colors, real icons, real type scale. If she's comfortable in Figma and you want native inline comments, Option A is the gold standard. Option C is the fallback if simplicity matters most.
